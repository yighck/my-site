After working with Next.js for a while, here are some practical tips that have significantly improved my development workflow.

## 1. Use Server Components by Default

In the App Router, components are Server Components by default. Only add `"use client"` when you need interactivity (state, effects, event handlers).

```tsx
// ✅ Server Component (default) — fast, no JS shipped to client
export default async function PostList() {
  const posts = await db.posts.findMany();
  return (
    <ul>
      {posts.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

```tsx
"use client";

// ✅ Client Component — only when you need interactivity
import { useState } from "react";

export function LikeButton() {
  const [liked, setLiked] = useState(false);
  return (
    <button onClick={() => setLiked(!liked)}>
      {liked ? "❤️" : "🤍"}
    </button>
  );
}
```

## 2. Colocate Files with Routes

Keep related files together. Put styles, tests, and helpers next to the route they belong to:

```
app/
  blog/
    [slug]/
      page.tsx        ← route component
      loading.tsx     ← loading state
      error.tsx       ← error boundary
      opengraph-image ← OG image
```

## 3. Use `loading.tsx` for Instant Feedback

Add a `loading.tsx` file to any route folder to get automatic loading UI with Suspense:

```tsx
// app/blog/[slug]/loading.tsx
export default function Loading() {
  return <div className="animate-pulse">Loading post...</div>;
}
```

## 4. Parallel Data Fetching

Fetch data in parallel to avoid waterfalls:

```tsx
export default async function Dashboard() {
  // These run in parallel!
  const [user, posts, analytics] = await Promise.all([
    getUser(),
    getPosts(),
    getAnalytics(),
  ]);

  return (
    <div>
      <UserHeader user={user} />
      <PostList posts={posts} />
      <AnalyticsChart data={analytics} />
    </div>
  );
}
```

## 5. Metadata for SEO

Use the `metadata` export for dynamic SEO:

```tsx
import type { Metadata } from "next";

export async function generateMetadata({ params }): Promise<Metadata> {
  const post = await getPost(params.slug);
  return {
    title: post.title,
    description: post.description,
    openGraph: {
      title: post.title,
      description: post.description,
    },
  };
}
```

These patterns have made my Next.js apps faster and easier to maintain. Hope they help you too!
