本指南介绍如何向站点添加文档页面。

## 目录结构

文档存放在 `content/docs/` 目录下，可以组织为子目录：

```text
content/docs/
  getting-started/
    introduction.mdx
    installation.mdx
  guides/
    writing-blog.mdx
    adding-docs.mdx
```

子目录在侧边栏导航中显示为**分组**。

## 文件命名

与博客文章相同的格式：

```text
{slug}.mdx
```

URL 路径镜像目录结构：

- `content/docs/getting-started/installation.mdx` -> `/docs/getting-started/installation`

## Frontmatter

```yaml
---
title: "页面标题"
date: "2026-06-19"
description: "页面的简要描述。"
tags: ["可选", "标签"]
---
```

## 侧边栏导航

侧边栏根据目录结构**自动生成**：

- **目录**成为可折叠的分组
- **文件**成为链接，按文件系统顺序排列
- frontmatter 中的 **title** 作为链接文本显示

要控制页面顺序，可以使用数字前缀命名文件：

```text
01-introduction.mdx
02-installation.mdx
03-configuration.mdx
```

## 编写技巧

### 使用标题构建结构

标题（`##`、`###`）会自动生成页面右侧的目录。

### 添加代码示例

技术文档需要具体的代码示例：

```bash
# 在 bash 块中展示命令
npm run build
```

```tsx
// 在对应语言块中展示代码
export default function Page() {
  return <div>内容</div>;
}
```

### 使用引用块标注重要信息

带粗体前缀的引用块适合做提示：

> **注意：** 这是读者应该关注的重要提示。
>
> **警告：** 修改此配置时请谨慎操作。

### 页面间互相引用

使用相对链接引用其他文档页面：

```markdown
请参阅[安装指南](/docs/getting-started/installation)了解搭建步骤。
```

## 单语内容

本站现在只保留中文内容。新增文档时只需要创建一个 `.mdx` 文件。
